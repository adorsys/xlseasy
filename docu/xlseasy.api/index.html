xlseasy api

xlseasy.api implements the programming interface and is used as component to communicate with xlseasy.cbe and xlseasy.impl.

Main parts

  • annotation
  • boot
  • utils

It contains a sub-part named filter. This part implements the annotation manipulation by filtering and annotation's system exceptions handling.

annotation consists essentially of rewrited Apache POI classes (such as Workbook , Sheet , ...) using to create and manipulate spreadsheets.

The part boot contains essentially Inspector classes to examine fields and styles for given classes, elements or values.

It also contains workbook's configurator ( WorkBookSheet , WorkbookCbe ) and a spreadsheet service starter named SpreadSheetServiceBootStrap.

The class WorkbookCbe contains the list of sheet descriptions and should be used to build the workbook and its fields.

The class SpreadSheetServiceBootStrap is called first during the test execution to create the basis of the application programming interface (api).

It contains among others the following classes:

  • Assert: arguments validator using to identify programming errors early and clearly at runtime.
  • CollectionFieldFilter: matches all fields that are collection types.
  • FieldHelper: helps to build setter and getter Methods from a given name.
  • ReflectionUtils: works with the reflection API and handling reflection exceptions.
  • XlseasyUtils: handles errors and system exceptions from sheet.
